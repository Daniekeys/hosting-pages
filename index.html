HOW TO UPLOAD AND HOST YOUR WEBSITE ON GITHUB

We are going to learn step by steps by which we can take inorder to host our webpages from scratch.

STEPS TAKEN

DOWNLOAD GITHUB DESKTOP

Just go to google and search for github desktop. After which you will select the operating system your system is using then click download. Once the download process is over kindly install it on your PC. OR Download here

CREATE A NEW REPOSITORY

After the installation process is successful , navigate to the top left corner of the screen and click on file underneath the file you would see New repository as one of the options listed. click on New Repository.

FILL IN YOUR DETAILS

A page will pop-up once you click on create new repository . Fill in the Name and Description field approprately. Then select the choose button to select the folder that containers your file. (Note: the html file must be renamed to index so that the browser can easily interprete it.)

SELECT THE FOLDER

A page will pop-up once you click on choose button kindly navigate to the folder that houses your file and select the folder. For example, we are picking a folder named Marvin

CREATE REPOSITORY

After the folder has been selected just leave the remaining options as default i.e the gitignore, licence should be left at None. There after click on create Repository

PUBLISH REPOSITORY

After the repository has been created we will need to publish it to github. so click on Publish repository

PUBLISH REPOSITORY 2

A page will be popped out, fill in the required field and make sure keep code private is not selected (it should only be selected if you dont want others to see your source code) then click on Publish repository.

SUCCESSFULLY PUBLISHED

Once it has been published successful the top navbar item should display last fetched just now . Check the image below for clearity

GITHUB FOLDER CREATED

Once everything from the previous stage is all set all you need to do is to navigate back to the folder that contains your file. on getting there a new folder would have been created and it will be named the initial name of your newly created repository. Just copy all files you see below it . In our use case we only have index.html and styles.css files so thats what we will copy . (Remember that ist very much safer if you only copy and not cut the files so that once you run into trouble you can always have a backup ready)

COPY AND PASTE

Paste the copied files inside the folder created by github. In our use case here its pasted inside the testing folder. (If you are not sure if the folder is created by github just check if the folder has git attribute file inside once it does certainly its the one).

LOCAL CHANGES DETECTION

Open your Github desktop you should see the below displayed on your screen. That means Github has detected a change in the folder and the changes must be committed to the master.

COMMIT TO MASTER

Check the buttom left side of your screeen you should see a small page . Fill in the summary and description field and click on the commit to master button.

PUSH TO ORIGIN

Click on the push to origin button to update the previous file base.

LOCATE YOUR NEW REPOSITORY

Login to your github account using the browser then look for your repository incase you are having trouble navigating there just search for github.com/username the username is your github username E.g github.com/daniekeys

IDENTIFY AND OPEN THE REPOSITORY YOU WANT TO USE

Select the repository you want to host as a github page

SETTINGS

Locate the settings icon its among the functions listed horizontally on the top.

GITHUB PAGES

Inside the settings page scroll down till you get to Github pages has higlighted in the picture below then click on check it out here.

GITHUB PAGES SETTINGS

Under the Source you will click on the first box which is Branch (by default its set to None) click on it and change it to main. Then click save. Leave all other settings as defaults .

GITHUB PAGE IS READY

You should see (Your site is ready to be published ) the page will contain a link to your page . Note that you will have to wait for like 3mins max for the effect to reflect, it all depends on the server thats why I recommened you click on the link exactly 3 mins later .
